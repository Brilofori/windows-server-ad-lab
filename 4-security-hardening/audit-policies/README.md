Audit Policies (Workstation Baseline)

This directory documents all audit configurations applied under the Workstation Baseline GPO. These settings ensure that important security-relevant events are logged for detection, investigation, and incident response.

Implemented Audit Categories
1. Account Logon
Logs authentication events that validate user credentials on domain controllers.

2. Logon/Logoff
Tracks workstation login, logout, lock, unlock, and session disconnect events.

3. Object Access
Logs access attempts to key system resources, protected files, and registry locations.

4. Policy Change
Logs modifications to user rights assignments, audit policies, and security settings.

5. Account Management
Captures events related to user, group, and computer account creation, deletion, and modifications.

Purpose
These audit policies provide visibility into authentication activity, administrative changes, and sensitive resource access.
They support:
Threat detection
Post-incident investigation
Compliance requirements
Monitoring for privilege misuse
All settings follow recommended enterprise auditing baselines.
